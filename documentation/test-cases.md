# Test Cases — Calendar Widget Generator
## 1. Scope

Тестируется страница генерации iframe-кода календаря мероприятий:

***Покрытие:***

- Функциональная логика генерации iframe

- Валидация размеров

- Синхронизация элементов управления

- Передача параметров в src

- Копирование кода

***Не покрывается:***

- Внутренний контент iframe (внешний сервис)

- Кроссбраузерность

- Визуальная пиксель-перфект проверка

## 2. Приоритизация

***P1*** — критично для основной функции генерации

***P2*** — важно, влияет на UX и корректность состояния

***P3*** — граничные и дополнительные проверки

## 3. Test Cases
### **TC-01 Генерация iframe с дефолтными значениями**

***Priority***: P1
***Automation***: Yes

***Preconditions***: Страница загружена

***Steps***:

1. Не изменяя настройки, нажать "Сгенерировать превью"

***Expected result***:

превью и код iframe отображается

***В сгенерированном коде***:

- присутствует id

- корректный src

- width соответствует значению по умолчанию

- height соответствует значению по умолчанию

- frameborder="0"

### **TC-02 Изменение ширины через input**

***Priority***: P1
***Automation***: Yes

***Steps***:

1. Ввести новое валидное значение ширины (значения от 230 до 1020)

2. Нажать "Сгенерировать превью"

***Expected result***:

- width обновлён в коде iframe

- width превью соответствует введённому значению

### **TC-03 Граничные значения ширины - негативные проверки**

***Priority***: P2
***Automation***: Yes

***Steps***:

1. Ввести значение ширины меньше минимального (от 0 до 229 включительно) и нажать "Сгенерировать превью".

2. Ввести значение ширины больше максимального (от 1021 до 99999) и нажать "Сгенерировать превью".

***Expected result***:

- Если введено < 230 → поле и iframe автоматически устанавливают width = 230.

- Если введено > 1020 → поле и iframe автоматически устанавливают width = 1020.

- iframe и превью корректно обновляются

### **TC-04 Синхронизация slider и input (ширина)**

***Priority***: P2
***Automation***: Yes

***Steps***:

1. Изменить ширину через slider

2. Проверить значение в input

3. Изменить input

4. Проверить позицию slider

***Expected result***:

- Полная синхронизация в обе стороны

### **TC-05 Опция "на всю ширину контейнера"**

***Priority***: P1
***Automation***: Yes

***Steps***:

1. Активировать опцию

2. Сгенерировать превью

***Expected result***:

- width="100%" (или соответствующее значение)

- input и slider недоступны

### **TC-06 Выбор цветовой темы**

***Priority***: P1
***Automation***: Yes

***Steps***:

1. Выбрать другую цветовую тему (не дефолт- turquoise)

2. Сгенерировать превью

***Expected result***:

- В src обновлён параметр theme

- превью перезагружается с новой темой

### **TC-07 Выбор страны**

***Priority***: P1
***Automation***: Yes

***Steps***:

1. Выбрать конкретную страну

2. Сгенерировать превью

***Expected result***:

- В src присутствует параметр event_country

- Значение корректно передано

### **TC-08 Выбор тематики**

***Priority***: P1
***Automation***: Yes

***Steps***:

1. Выбрать тему

2. Сгенерировать превью

***Expected result***:

- В src обновлён параметр в значении event_group=

### **TC-09 Отмена выбора тематики**

***Priority***: P1
***Automation***: Yes

***Steps***:

1. Выбрать любую тему из списка

2. Сгенерировать превью

3. Проверить обновленный параметр event_group в коде iframe

4. Отменить выбор темы

5. Сгенерировать превью

***Expected result***:

- В src актуальный параметр event_group

### **TC-10 Копирование кода**

***Priority***: P1
***Automation***: Yes

***Steps***:

1. Сгенерировать код

2. Нажать "Скопировать код"

3. Получить данные из clipboard

***Expected result***:

- Буфер содержит точно отображаемый HTML


## 4. Обоснование автоматизации

***Автоматизированы сценарии***:

- Повторяемые

- Критичные для бизнес-логики

- Имеющие явный ожидаемый результат

- Подверженные регрессии (валидация, синхронизация)

***Не автоматизированы***:

- Проверки установки высоты контейнера (дублируют логику установки ширины, не критично для тестового задания)

- Console errors (контролируется в процессе P1/P2)

- Смена языка страницы (RU / EN)

- Визуальная эстетика

- Тексты инструкций

- Проверка рекомендаций по вставке кода

## 5. Out of Scope

В рамках данного тестового проекта не автоматизировано:

- Смена языка страницы (Русский / English)

- Кроссбраузерное тестирование

- Pixel-perfect визуальное сравнение

- Accessibility-тестирование

- Performance-тестирование

- Нагрузочное тестирование

- Проверка SEO и мета-тегов

- Проверка рекомендаций по размещению iframe в HTML-документе

**Причины**:

- iframe загружает внешний ресурс

- фокус проекта — логика генерации

- ограниченный объём тестового задания
